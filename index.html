What I have Learned

Throughout my short time here, I have learned some valuable lessons on accessibility. So, I will share what I have learned here.


POUR Principles

I have already written something for this so I will just leave a link for that: POUR Principles

PowerPoints

When creating a PowerPoint use a consistent and organized layout for each slide. Avoid cluttered designs and ensure sufficient spacing. Also, use a bigger font size to ensure readability, especially when presenting in-person or on larger screens. The larger font accommodates various viewing distances and helps individuals with visual impairments.

Word Documents

For Word documents using headers helps individuals using screen readers to navigate through the document easily. You can use the built-in heading styles in Word, such as, Heading 1, Heading 2, etc., to structure your document. To apply a heading style, select the text and choose the appropriate heading style from the “Styles” dropdown menu in the “Home” tab at the top of the screen.

PDFs

Tagging the PDF enhances its accessibility. Tags provide a structured representation of the content, allowing assistive technology to interpret and navigate the document more effectively. Tags establish the reading order of the document. The reading order should match the visual order to ensure content is presented correctly. So, it is important to have the document tagged correctly. Using Adobe Acrobat Pro, you can use their built-in accessibility checker to help check your documents. These checks are designed to identify potential issues and provide suggestions for improving accessibility.

Videos

If your material contains multimedia elements like videos or audio files, provide captions or transcripts. This makes the content accessible to people with hearing impairments or those who cannot access the media. You can provide a separate transcript in the document.

Websites

Headings

Using the correct heading tags(h1 the biggest to h6 being the smallest) is important because if not correctly used a screen reader might assume there is content missing. It is best to use h1 for the main title, h2 for heading that come after the title, h3 for subheadings, and so on.

Aria-Labels

Aria- labels are used to provide a text alternative when the visible label or text associated with an element is insufficient or unavailable. They help screen readers and other assistive technologies convey the intended meaning of an element to users who may not be able to see it. When labeling elements, it’s generally recommended to use visible text labels whenever possible. Aria-labels should be used as a fallback option or when the visible label is not available. This ensures that users who can see the content can benefit from the visual cues.

Syntax: Aria-labels can be applied using the “aria-label” attribute, which is added to the HTML element you want to label. The value of the attribute should be concise and descriptive text that describes the purpose or function of the element.

Keyboard Functionality

Keyboard functionality is an essential part of website accessibility, as it allows individuals who cannot use a mouse or other pointing device to navigate and interact with web content using only the keyboard.

Ensure that interactive elements, such as links, buttons, and form inputs, can receive focus using the “Tab” key. Each focusable element should have a clear and visible focus indicator, such as a highlight or outline, to indicate the currently focused element. Make sure that the keyboard navigation order follows a logical and meaningful sequence. Users should be able to navigate through interactive elements and content in a way that makes sense and follows the visual layout of the page
